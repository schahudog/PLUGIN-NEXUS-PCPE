## [1.5.1] — 22/06/2026 — Correção da Skill `nexus-skill-creator`: Injeção Obrigatória no Plugin

### Motivação
A versão 1.5.0 da skill `nexus-skill-creator` estava incompleta: ao criar uma nova skill, ela a
empacotava como arquivo `.skill` isolado, entregando-a apartada do plugin. O comportamento correto
é injetar a nova skill diretamente no NEXUS e entregar um `nexus-pcpe.plugin` atualizado e
pronto para reinstalação — sem exigir etapa manual adicional do usuário.

Adicionalmente, o caminho do plugin NEXUS estava hardcoded com IDs de sessão e de plugin da sessão
de criação, tornando a skill inutilizável em sessões subsequentes.

### Bugs Corrigidos

#### Bug 1 — Entrega de `.skill` isolado em vez de `.plugin` atualizado (`nexus-skill-creator/SKILL.md`, Seção 8)
- **Causa raiz:** A Seção 8 instruía empacotar a nova skill como `.skill` (zip renomeado) e
  entregá-la ao usuário sem injetar no plugin NEXUS.
- **Correção:** Seção 8 reescrita integralmente como **"Injeção no NEXUS (Fase Obrigatória)"**.
  O fluxo agora: (1) localiza o plugin NEXUS instalado, (2) copia para `/tmp`, (3) injeta a skill,
  (4) incrementa versão patch, (5) empacota como `nexus-pcpe.plugin`, (6) entrega ao Lukas.
- **Impacto:** Toda skill criada via `nexus-skill-creator` é entregue já integrada ao plugin,
  pronta para reinstalação imediata — sem arquivo `.skill` orphan.

#### Bug 2 — Path do plugin NEXUS hardcoded com IDs de sessão (`nexus-skill-creator/SKILL.md`, Seção 8)
- **Causa raiz:** Os comandos bash usavam caminhos literais com IDs de sessão e de plugin
  (`/sessions/lucid-beautiful-cerf/...`, `plugin_01UkBA1Lp6w8TiZU8g2BveMw`) que mudam a cada
  sessão do Cowork.
- **Correção:** Substituído por descoberta dinâmica via busca no `plugin.json`:
  ```bash
  NEXUS_SRC=$(find /sessions/ -name "plugin.json" 2>/dev/null \
    | xargs grep -l '"nexus-pcpe"' 2>/dev/null \
    | head -1 \
    | sed 's|/.claude-plugin/plugin.json||')
  OUTPUTS=$(find /sessions/ -maxdepth 5 -name "outputs" -type d 2>/dev/null | head -1)
  ```
- **Impacto:** A skill funciona corretamente em qualquer sessão futura, independente de IDs gerados.

### Arquivos Modificados
- `skills/nexus-skill-creator/SKILL.md` — Seção 6 (Fase 5 renomeada) e Seção 8 reescrita; versão 1.0.0 → 1.1.0
- `.claude-plugin/plugin.json` — versão 1.5.0 → 1.5.1
- `CHANGELOG.md` *(este arquivo)*

## [1.5.0] — 22/06/2026 — Nova Skill: `nexus-skill-creator`

### Motivação
O NEXUS não possuía um mecanismo interno para criar novas habilidades específicas ao contexto
policial. A criação de skills dependia de ferramentas genéricas (skill-creator da Anthropic) que
não conhecem os padrões institucionais do plugin: clonagem BytesIO, estilos Heading 1/2 nativos
para Google Docs, modelos .docx com brasões, guardrails anti-alucinação policiais, ou a taxonomia
de agentes (Delegado, Escrivão, Investigador, Perito, Revisor).

### Nova Funcionalidade

#### `nexus-skill-creator` — Ateliê de construção do NEXUS
Skill especializada em criar novas habilidades para o plugin, com conhecimento completo dos padrões
institucionais. Aciona-se quando o usuário pedir para criar uma skill para uma demanda recorrente,
complexa ou especializada que não existe ainda no NEXUS.

**Protocolo de entrevista estruturado em 3 blocos:**
- Bloco A — O que a skill faz (nome, entradas, output, estrutura)
- Bloco B — Contexto investigativo (crime, sistemas, base normativa, guardrails especiais)
- Bloco C — Integração com o NEXUS (agente natural, conflitos com skills existentes)

**Taxonomia de classificação de skills por tipo:**

| Tipo | Produz | Modelo base | Agente natural |
|------|--------|-------------|----------------|
| Peça Cartorária | .docx | modelo_raf.docx | Escrivão |
| Análise Investigativa | texto ou .docx | modelo_raf.docx | Investigador / Perito |
| Produto de Inteligência | texto estruturado | — | Perito |
| Diagnóstico | análise + recomendação | — | Delegado |
| Análise de Dados | texto + tabelas | modelo_raf.docx | Perito |
| Peça Cautelar | .docx | modelo_raf.docx | Delegado |

**Padrões obrigatórios embutidos** — a skill conhece e aplica automaticamente:
- Clonagem BytesIO (sem `shutil.copy2`)
- Heading 1/2 nativos com spacing e keepNext/keepLines desabilitados
- Bordas de tabela via XML (sem `Table Grid`)
- Guardrails anti-alucinação policiais
- Bloco de assinatura centralizado

## [1.4.8] — 22/06/2026 — Correção Crítica do Gerador de RAF e Atualização do Modelo Institucional

### Motivação
A skill `analise-rif` falhava com `PermissionError` ao gerar o RAF em qualquer instalação limpa do plugin.
O arquivo `resources/modelo_raf.docx` é armazenado com permissão somente leitura (`-r-x------`) pelo
instalador do Cowork. O código anterior usava `shutil.copy2(modelo, destino)`, que copia as permissões
da fonte para o destino — tornando o arquivo de saída também read-only. Quando `python-docx` tentava
salvar (`doc.save(output_path)`), o `ZipFile` abria em modo escrita e disparava `PermissionError`.

Adicionalmente, o modelo `modelo_raf.docx` anterior não era o modelo institucional definitivo
(sem brasões visíveis). O usuário forneceu o modelo correto com cabeçalho completo (brasões PCPE +
brasão PE + marca d'água), que foi incorporado ao plugin.

### Bugs Corrigidos

#### Bug 1 — `PermissionError` ao salvar RAF (`analise-rif/SKILL.md`, seção 2.3)
- **Causa raiz:** `shutil.copy2(modelo_path, output_path)` → arquivo destino herda `-r-x------`
  → `Document(output_path)` abre OK (leitura) → `doc.save(output_path)` falha (ZipFile 'w' mode)
- **Correção:** Substituído por padrão BytesIO seguro:
  ```python
  import io
  with open(modelo_path, 'rb') as f:
      src_bytes = f.read()
  doc = Document(io.BytesIO(src_bytes))   # modelo em memória — sem tocar permissões
  # ... modificar conteúdo ...
  buf = io.BytesIO()
  doc.save(buf)
  buf.seek(0)
  with open(output_path, 'wb') as f:      # grava binário com permissão padrão do processo
      f.write(buf.read())
  ```
- **Impacto:** Eliminado definitivamente; funciona em qualquer instalação independente de permissões.

#### Bug 2 — `KeyError: "no style with name 'Table Grid'"` (`analise-rif/SKILL.md`, seção 2.3)
- **Causa raiz:** O `modelo_raf.docx` não contém o estilo `Table Grid` — apenas `Normal Table` e
  `TableNormal`. Código anterior usava `tbl.style = 'Table Grid'` causando `KeyError` e abortando
  toda a geração.
- **Correção:** Adicionado helper `adicionar_bordas_tabela()` que injeta bordas via XML direto
  (`w:tblBorders`), sem depender de estilo nomeado.
- **Nota:** Adicionada diretiva `⚠️ CRÍTICO` no SKILL.md proibindo `tbl.style = 'Table Grid'`.

### Atualização de Recurso

#### `analise-rif/resources/modelo_raf.docx` — Substituição pelo modelo institucional definitivo
- **Antes:** Modelo anterior sem brasões visíveis no cabeçalho
- **Agora:** Modelo institucional definitivo fornecido pelo usuário, contendo:
  - Brasão PCPE (esquerda)
  - Brasão do Estado de Pernambuco (direita)
  - Marca d'água institucional (behind text)
  - Estilos nativos `Heading 1` e `Heading 2` para navegação no Google Docs

### Arquivos Modificados
- `skills/analise-rif/SKILL.md` — Seção 2.3 reescrita (BytesIO + bordas XML); versão 2.0.0 → 2.1.0
- `skills/analise-rif/resources/modelo_raf.docx` — Substituído pelo modelo institucional definitivo

## [1.3.0] — 21/06/2026 — Correção de Bugs na Geração de Documentos .docx

### Bugs Corrigidos

#### Bug 1 — Limpeza do body com iteração direta (`comunicacao-servico/references/codigo_geracao.md`)
- **Problema:** `gerar_cs_estruturada()` iterava diretamente sobre `doc.paragraphs` e `doc.tables`
  enquanto removia elementos (`p._element.getparent().remove(...)`). Em Python, modificar uma lista
  durante a iteração provoca saltos silenciosos — elementos do modelo vazavam para o documento final.
- **Correção:** Substituído por iteração sobre lista congelada via `list(body)` e remoção direta
  dos nós filhos do `body`, incluindo `sdt` (structured document tags).

#### Bug 2 — Limpeza do body com iteração direta (`comunicacao-servico/references/codigo_geracao_narrativa.md`)
- **Problema:** Mesmo padrão do Bug 1 em `gerar_cs_narrativa()`.
- **Correção:** Idêntica ao Bug 1.

#### Bug 3 — Estilo `'Table Grid'` inexistente no modelo (`relatorio-final/SKILL.md`)
- **Problema:** A instrução `tbl.style = 'Table Grid'` causava `KeyError` e abortava toda geração
  do RFIP. O arquivo `modelo_relatorio_final.docx` contém apenas os estilos `'Normal Table'`
  e `'TableNormal'`.
- **Correção:** Removida a atribuição de estilo por nome. Adicionada nota `⚠ ATENÇÃO` na seção 3.4
  do SKILL.md proibindo uso de `tbl.style` e determinando bordas via `set_cell_borders()` (XML direto).

#### Bug 4 — Clone via `shutil.copy2` para o arquivo de destino (`portaria-ip/SKILL.md`, ambos os arquivos de código)
- **Problema:** O fluxo `shutil.copy2(modelo, dest) → Document(dest) → doc.save(dest)` falha com
  `PermissionError` quando o arquivo de destino já existe e está travado por outro processo
  (segunda execução, sincronização OneDrive, antivírus).
- **Correção:** Substituído por carregamento do modelo em memória via `io.BytesIO`:
  ```python
  import io
  with open(modelo_path, 'rb') as f:
      buf = io.BytesIO(f.read())
  doc = Document(buf)
  # ... modificar ...
  doc.save(output_path)  # grava direto no destino sem conflito
  ```
